@ 인터넷이란?
-> 전 세계의 컴퓨터가 서로 연결되어 정보를 교환할 수 있는 하나의 거대한 컴퓨터 통신망
-> 인터넷에서는(인터넷 기반) 다양한 서비스를 제공함
 - Web Service, FTP Service, Mail Service 등등
-> 프로토콜을 이용하여 통신규약을 약속하여 통신함
 - Web -> HTTP, Mail -> SMTP, POP3, IMAP

(프로토콜 = http(암호화x), https(암호화o), ftp(파일서버))

@ 웹(Web)이란?
-> 1989년 CERN(유럽 입자 물리 연구소)의 팀 버너스 리에 의해 개발 됨
-> Hyper Text를 통해 정보를 교환하는 서비스
 - Hyper Text : 하나의 웹 문서에서 제공하는 연결 주소
   (링크)를 통해 다른 문서로 이동하는 것.
-> 네트워크에 연결된 시스템의 "정보"를 한 위치에서 "검색"할 수 있는 서비스

@ Web Version
 - Web 1.0 : 정적인 검색 서비스 제공 (초기 서비스 버전),
	     동적인 검색 서비스 제공 (현재 가장 많이 사용되는 기본 서비스 버전),
	     동적인 검색 및 사용자의 참여, 공유, 개방
 - Web 2.0 : 지능형 웹 서비스 (facebook, youtube, instagram, ..),
	     사용자에게 필요한 지식을 자동으로 검색하여 제공

@ HTML (Hyper Text Markup Language)
-> Web Data(Web Page)를 전송하기 위해 사용되는 HyperText를 구성하는 Markup Language
   (마크업 언어)
-> Hyper Text(하이퍼 텍스트) : Hyper Link를 통해 구성된 다른 정보로 쉽게 연결시켜주는
   구조화된 문서
-> Markup Language(마크업 언어) : 태그 등을 이용하여 문서나 데이터의 구조를 정의하는 언어

@ HTML의 역사
1. 인터넷의 시작
 - 핵과 같은 공격에 대비하여 전체 통신 시스템에서 데이터를 안전하게 보관 및 전송할 수 있는
   시스템을 만듦
 - ARPANET(Advanced Research Projects Agency)

2. 유럽 공동 원자핵 연구소(CERN)의 팀 버너스 리
 - 웹을 개발
 - 웹 표준을 지정하는 기관(HTML 표준을 제정하는 단체) W3C재단을 창설
 - 웹 브라우저 전쟁이 일어나게 됨

3. 제1차 웹 브라우저 전쟁
 - 최초의 GUI 모습을 갖춘 '모자이크' 브라우저가 탄생
 - 모자이크는 발전하면서 모자이크 커뮤니케이션에서 넷스케이프 커뮤니케이션으로 이름을
   바꾸고 브라우저 명칭도 넷스케이프로 바꿈
 - 이에 맞서 MS가 인터넷 익스플로러를 발표함
 - 넷스케이프와 경쟁에서 밀린 인터넷 익스플로러는 윈도우 운영체제에 강제적으로 설치를 해서
   점유율을 높임

* 플러그인
 - 웹 브라우저 전쟁으로 인해 웹은 엄청난 속도로 발전함
 - 하지만 웹 표준을 지정하는 W3C는 빠르게 대응하지 못함
 - 이에 불만을 느낀 기업들은 플러그인을 만듦
-> 플러그인은 웹 브라우저와 연동되는 특정 프로그램을 사용자 PC에 추가로 설치해 웹 브라우저
   기능을 확장하는 방법
 - 1996년부터 현재 어도비 플래시, 액티브엑스 등 웹을 점점 풍부한 기능을 가진 공간으로
   만들어줌
 
* 웹 2.0 시대
 - 2000년대부터 엑티브엑스를 기반으로 웹 어플리케이션이 제작되었음
 - 또한 플래시 기반으로 일반 사용자를 위한 애니메이션들이 만들어졌음 (졸라맨, 마시마로, ..)
 - 플래시와 액티브엑스는 점점 발전하게 됨
 - 인터넷은 점점 사용자끼리 뭉치고 새로운 콘텐츠를 개발할 수 있는 공간으로 발전하게 되고
   정보 및 콘텐츠를 공유하며 웹 2.0시대를 맞이하게 됨
-> YOUTUBE, 위키피디아, ...

* WHATWG의 등장
 - 전 세계 최고 점유율을 가진 인터넷 익스플로러는 사실상 W3C의 표준 웹 브라우저가 되고
   모든 웹 사이트에 액티브엑스 플러그인이 들어가게 되면서 웹사이트는 무거워지기 시작함
   (시작하기 전에 액티브엑스를 설치해야 하기 때문)
 - 이를 막고자 2004년 W3C 회의에서 모질라 재단과 오페라 소프트웨어는 새로운 HTML 표준
   제안서를 제출함
 - WHATWG(Web Hypertext Application Technology Working)
   WHATWG는 인터넷 익스플로러를 제외한 웹 브라우저
   제공 기업에서 설립한 새로운 웹 표준 기관
 - WHATWG는 새로운 웹 표준으로 Web Application 1.0, XHTML 2.0 등등의 새로운 웹표준을 만들고
   W3C에서 Web Application 1.0을 표준으로 선택함
 - Web Application 1.0 표준을 HTML5표준으로 변경됨
 - HTML5 표준은 2012년에 초안을 발표하고 2014년 10월 정식 권고안을 발표하였음.

@ HTML5의 특징
-> 구조적 설계 지원 (시맨틱)
* 시맨틱 : 이름만 보고도 역할을 짐작할 수 있는 것
-> 그래픽 및 멀티미디어 기능 강화
-> CSS3 / Jvascript 지원
-> 다양한 API 제공
-> 모바일 웹 지원 / 장치 접근 가능 (배터리 정보, 카메라, GPS 등)

4. 제2차 웹 브라우저 전쟁
-> 다른 웹 브라우저는 모두 최신 표준을 지원하는데 인터넷 익스플로러는 오히려 최신을 지원하지
   못하는 역현상이 발생함
-> 지금까지의 웹 브라우저 점유율을 뒤집을 수 있는 기회들이 생겨나고 모든 웹 브라우저가
   빠른 속도로 업데이트를 함
-> 모질라의 파이어폭스는 3년에 한번 하던 버전 업을 2단 단위로, 구글의 크롬은 10주단위로
   버전 업이 되고 있음.
-> 현재까지도 웹 브라우저 전쟁이 지속되고 있고 앞으로 우리가 겪어야 함

@ HTML 구성요소

<p align = 'center'> HTML/CSS </p>

<p : 시작태그
align : 속성
'center' : 속성값
HTML/CSS : 내부문자
</p> : 종료 태그

(속성이랑 속성값은 없어도 됨)

태그(Tag) : '<'와'>'로 묶인 명령어, 시작태그(<태그>)와 종료태그(</태그>)를 한 쌍으로 이용

요소 : 시작태그와 종료태그로 이루어진 모든 명령어
하나의 HTML 문서는 요소들의 집합

속성 : 요소의 시작태그에만 사용 / 명령어 구체화 역할
여러개의 속성을 사용할 수 있으며 속성의 구분은 공백

변수 / 속성값 : 속성이 가지는 값, 값 입력시 "" 혹은 ''를 이용

@ HTML 주의사항
-> 태그는 대소문자를 구분하지 않음, 소문자 권장
-> 시작태그 시작하면 반드시 종료태그로 종료
-> 파일 확장자는 반드시 html(htm)로 설정할 것
-> 문자의 공백은 한 개만 인식하므로 공백을 추가하기 위해서는 특수기호를 이용해야 한다.
   -> &nbsp;

@ 글자 태그

1. h 태그
<h?> </h?>
-> 제목을 입력할 때 사용하는 태그, 폰트의 크기가 태그에 따라 정해져 있음
-> h1 ~ h6 까지 폰트의 크기가 작아짐

2. br 태그
-> 문장에서 줄 바꾸기(개행)할 때 사용

3. hr 태그
-> 페이지에 가로로 밑줄을 만들어 줄 때 사용

4. p 태그 <p> </p>
-> 한 개의 단락을 만들 때 사용
-> 자동으로 문장 개행 후 한줄 공백을 만들어 줌

5. pre 태그 <pre> </pre>
-> 입력한 그대로 출력할 때 사용
-> 띄어쓰기, 들여쓰기, 줄 바꿈이 입력됨 그대로 출력됨

6. strong태그 또는 b태그
-> 문장에서 문자를 강조하여 굵게 표시할 경우에 사용

7. em 또는 i 태그
-> 문자에 기울임을 주는 태그

8. blockquote 태그
-> 다른 블로그나 사이트의 글을 인용할 경우 사용
-> 자동 들여쓰기가 되어 다른 텍스트와 구별할 수 있음

9. q태그
-> 다른 블로그나 사이트의 글을 인용할 경우 사용
-> 자동 들여쓰기가 되어 다른 텍스트와 구별할 수 있음
-> 인용문구에 " " 표시가 됨

10. mark 태그
-> 배경 부분이 노랑색이 되며 형광펜 표시가 된듯하게 출력



@ 목록 태그

1. <ul></ul>
-> 순서가 필요하지 않은 목록을 만들때 사용
-> 리스트 앞에 특정 모양이 출력됨(default 모양은 ㆍ)
-> 해당 모양은 CSS를 이용하여 수정할 수 있음

작성법
<ul>
	<li>내용1</li>
	<li>내용2</li>
	<li>내용3</li>
</ul>

2. <ol></ol>
-> 순서가 있는 목록을 만들때 사용
-> 속성으로 문자(abc... / ABC), 숫자(123...), 로마자 설정
-> CSS로 순서를 설정 가능

속성
type : 순서형식을 정하는 것(a,A,1,i)
start : 시작 순서

작성법
<ol type="설명문자" start="시작순서">
	 <li>내용1</li>
	 <li>내용2</li>
	 <li>내용3</li>
</ol>

3. <dl></dl>
-> 용어나 문장에 대한 정의(definition) 리스트를 만듦
-> 설정시 자동으로 들여쓰기가 됨

<dt></dt> : 정의의 제목
<dd></dd> : 정의의 내용

<dl>
	<dt> 정의제목 </dt>
	<dd> 정의내용 </dd>
	<dd> 정의내용 </dd>
</dl>
